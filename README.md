## 📋 Описание

Скрипт выполняет автоматическую настройку сервера под управлением **Ubuntu/Debian**, включая:

- 🔐 Защиту SSH (ключи, смена порта, отключение паролей)
- 🛡️ Настройку фаервола UFW
- 🚀 Оптимизацию сети (TCP BBR)
- 💾 Управление Swap-файлом
- 🔄 Автоматические обновления безопасности
- 🚫 Защиту от брутфорса (Fail2ban)
- 🌐 Cloudflare WARP (VPN/Proxy)
- 🎛️ Панель управления 3x-ui

Все действия логируются в `/var/log/server_setup.log`.

# ⚙️ Конфигурация

## 🔐 SSH настройки

> ⚠️ Если SSH-ключ не настроен, скрипт изменяет только порт.  
> Остальные параметры остаются по умолчанию, чтобы не потерять доступ к серверу.

| Параметр | Значение | Описание |
|----------|---------|----------|
| Порт | `2222` (по умолчанию) | Порт, на котором работает SSH (нестандартный порт снижает количество атак) |
| PermitRootLogin | `prohibit-password` | Запрещает вход под root по паролю, разрешает только по SSH-ключу |
| PasswordAuthentication | `no` | Полностью отключает вход по паролю |
| PermitEmptyPasswords | `no` | Запрещает вход с пустым паролем |
| PubkeyAuthentication | `yes` | Разрешает аутентификацию по SSH-ключам |
| MaxAuthTries | `3` | Максимальное количество попыток входа |
| LoginGraceTime | `30` | Время (в секундах) на ввод данных перед отключением |
| ClientAliveInterval | `300` | Интервал (в секундах) проверки активности клиента |
| ClientAliveCountMax | `2` | Количество пропущенных проверок перед разрывом соединения |

## 🛡️ Fail2ban настройки

Fail2ban — это защита от перебора паролей (брутфорса).  
Он отслеживает подозрительные попытки входа и автоматически блокирует IP-адреса.

---

### ⚙️ Основные настройки

```ini
[DEFAULT]
bantime = 1h          # На сколько блокируется IP (здесь: 1 час)
findtime = 10m        # За какой период считаются попытки (10 минут)
maxretry = 5          # Сколько ошибок допускается до бана
ignoreip = 127.0.0.1/8 ::1  # Белый список (эти IP никогда не блокируются)
```

📌 Проще говоря:  
Если за **10 минут** будет **5 неудачных попыток входа**, IP получит бан на **1 час**.

---

### 🔐 Защита SSH

```ini
[sshd]
enabled = true        # Включает защиту SSH
port = <ваш_порт>     # Укажите ваш SSH порт
logpath = /var/log/auth.log  # Лог, где ищутся попытки входа
maxretry = 3          # Всего 3 попытки до бана
bantime = 24h         # Бан на 24 часа
```

📌 Это значит:  
Если кто-то **3 раза неправильно введёт пароль/ключ**, он будет заблокирован на **24 часа**.

---

### 🚫 Защита от повторных атак (recidive)

```ini
[recidive]
enabled = true        # Включает усиленную защиту
logpath = /var/log/fail2ban.log  # Лог всех банов
filter = recidive     # Отслеживает повторные нарушения
action = iptables-allports[name=recidive]  # Блокирует доступ ко всем портам
bantime = -1          # Перманентный (вечный) бан
findtime = 24h        # Отслеживание за 24 часа
maxretry = 3          # Если забанен 3 раза — бан навсегда
```

📌 Это значит:  
Если IP уже банили **несколько раз за сутки**, он получает **вечный бан** на весь сервер.

---

### 💡 Итог

- Обычные атаки → бан на время  
- Повторные атаки → вечный бан  
- Ваш локальный IP → никогда не блокируется  

Это базовая, но очень эффективная защита сервера 🔒

# ❓ FAQ

### Можно ли запустить скрипт повторно?
Да, скрипт проверяет текущее состояние и пропускает уже настроенные компоненты.

---

### Как изменить порт SSH после установки?
Запустите скрипт повторно и выберите пункт изменения порта SSH.

---

### WARP не подключается, что делать?

Проверьте статус:
```bash
warp-cli status
```

Попробуйте перерегистрировать:
```bash
warp-cli registration new
```

---

### Где найти логин/пароль от 3x-ui?
После установки панель покажет данные в терминале.

Также можно использовать команду:
```bash
x-ui userinfo
```
